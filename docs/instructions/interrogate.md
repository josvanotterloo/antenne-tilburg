# Instruction: Interrogate Before Generating

Use the interrogatory pattern when asked to help design, plan, or document
something complex — especially when the relevant context lives in the user's
head rather than in writing.

## When to use

- New feature brainstorming (supplements `/superpowers:brainstorm`)
- Workshop or session design
- Stakeholder communication planning
- Any task where generating output immediately would require assumptions

## Rules

1. Do NOT generate the output document immediately.
2. Ask one question at a time.
3. Wait for the answer before asking the next question.
4. Keep questions specific — each one should unlock a concrete decision in
   the eventual output.
5. After 5–7 questions, summarize what you've learned and ask whether
   anything is missing or should be corrected.
6. Only then generate the output.

## Question discipline

- One question per message. No lists of questions.
- Start with the question that, if answered wrong, would invalidate everything
  else (audience, goal, constraint).
- Move from scope → constraints → content → tone.
- If an answer introduces a contradiction, surface it before continuing.

## Summary checkpoint (after question 5–7)

Before generating, write a short summary:

> Here's what I understand so far: [2–4 bullet points]. Anything missing
> or off?

Only proceed once the user confirms or corrects.

## Why

Generating output before understanding the context produces plausible-looking
work that needs to be re-done. The interrogatory pattern costs 5 minutes of
questions and saves a document rewrite.
